# Todolist 创建任务浮窗 - 重新设计说明

## 概述
已经完全重写了 Todolist 的创建任务浮窗 (TodolistCreateModal.vue)，提供了更清晰、更直观的任务创建流程。

## 主要功能

### 1. 四种独立的任务类别
浮窗顶部有四个大按钮，用户可以选择：
- **饮食** 🍽️ - 与饮食相关的任务
- **运动** 🏃 - 与运动健身相关的任务
- **睡眠** 😴 - 与睡眠相关的任务
- **自定义** ✏️ - 完全自定义的任务

### 2. 预设任务选择（非自定义类别）
选择类别后，会显示该类别的预设任务，用户可以快速选择：

**饮食预设**：
- 早餐 🥐
- 午餐 🍱
- 晚餐 🍜
- 零食 🍪
- 喝水 💧

**运动预设**：
- 慢跑 🏃
- 健身房 🏋️
- 瑜伽 🧘
- 游泳 🏊
- 散步 🚶

**睡眠预设**：
- 早睡 🛌
- 睡前准备 🧴
- 不玩手机 📵
- 冥想 🧘‍♀️
- 记录睡眠 📊

### 3. 任务详情表单
用户可以填写：
- **任务标题** - 必填，有针对性的占位符提示
- **任务描述** - 可选，提供更多细节
- **优先级** - 低/中/高，默认为"中"
- **日期选项** - 四种选项：

#### 日期选项说明
- **明天** - 任务截止日期设为明天
- **工作日** - 任务截止日期设为下一个工作日（周一至周五）
- **周末** - 任务截止日期设为下一个周末（周六或周日）
- **每天** - 任务标记为每日任务

### 4. 流程和样式
- **流程清晰**：逐步引导用户完成任务创建
- **样式一致**：与 Todolist 页面保持统一的视觉风格
- **响应式设计**：支持不同屏幕尺寸的适配

## 技术实现

### 前端组件
**文件**：`src/components/todolistView/TodolistCreateModal.vue`

**核心功能**：
- 状态管理：选中的类别、预设、表单数据
- 动态预设加载：根据选中类别显示对应预设
- 日期计算：根据选择的日期类型计算实际截止日期
- 表单验证：确保任务标题不为空

### 样式文件
**文件**：`src/css/components/TodolistCreateModal.css`

**特点**：
- 现代化的卡片式设计
- 平滑的过渡动画和悬停效果
- 梯度背景和阴影效果
- 响应式布局（支持平板和手机）

## 使用流程

1. **点击"新建任务"按钮** - 打开浮窗
2. **选择任务类别** - 四个选项之一
3. **选择预设任务**（如果非自定义）- 快速创建
4. **填写任务标题** - 必填项
5. **填写任务描述** - 可选项
6. **选择优先级** - 低/中/高
7. **选择日期** - 明天/工作日/周末/每天
8. **点击"创建任务"** - 提交

## 后端数据格式

创建的任务会包含以下字段：
```typescript
{
  title: string                    // 任务标题
  description?: string             // 任务描述
  priority: 'low' | 'medium' | 'high'  // 优先级
  category: 'diet' | 'exercise' | 'sleep' | 'custom'  // 类别
  category_icon: string            // 类别图标
  type: 'custom' | 'preset'        // 任务类型
  preset_type?: string             // 预设类型值
  is_daily: boolean                // 是否为每日任务
  date_type: string                // 日期类型
  due_date: string                 // 实际截止日期（YYYY-MM-DD）
}
```

- 平滑的过渡动画

## 响应式设计

### 桌面版 (≥1024px)
- 类别网格：4 列
- 预设网格：5 列
- 优先级和日期并排显示

### 平板版 (768px - 1023px)
- 类别网格：2 列
- 预设网格：3 列
- 优先级和日期并排显示

### 手机版 (<768px)
- 类别网格：2 列
- 预设网格：2 列
- 优先级和日期堆栈显示

## 验证和错误处理

- **标题验证**：如果标题为空，提交按钮自动禁用
- **错误提示**：任务创建失败时显示提示信息
- **加载状态**：创建中显示"创建中..."提示
