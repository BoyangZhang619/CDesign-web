================================================================================
                            一、页面功能概述
================================================================================

健康画像页面展示用户的综合健康状况，包括：
- 健康总体评分（圆形进度条）
- 三大维度评分：运动、饮食、睡眠
- 六维度雷达图分析：运动、饮食、睡眠、心肺功能、代谢指数、压力管理
- 身体指标卡片：BMI、心肺功能、代谢指数、睡眠质量
- 个性化健康建议列表
- 健康进度时间轴

页面在用户未完成健康档案设置时会强制显示设置弹窗。

================================================================================
                        二、主要 API 接口需求
================================================================================

【重要】所有请求都需要在 HTTP Header 中添加:
Authorization: Bearer <user_access_token>

### 1. 获取健康画像数据 (必需)
   
   端点: GET /api/health/portrait
   
   描述: 获取用户的完整健康画像数据
   
   请求头:
   {
     "Content-Type": "application/json",
     "Authorization": "Bearer <token>"
   }
   
   响应参数: 无
   
   【重要】响应格式使用嵌套结构: data.data.data
   
   前端访问数据路径: response.data.data (即后端返回的 data.data.data 部分)
   
   响应格式:
   {
     "success": true,
     "message": "获取成功",
     "data": {
       "data": {
         // 评分数据 (必需)
         "exerciseScore": 75,          // 0-100, 整数，运动评分。0表示无数据
         "mealScore": 68,              // 0-100, 整数，饮食评分。0表示无数据
         "sleepScore": 82,             // 0-100, 整数，睡眠评分。0表示无数据
         
         // 身体指标 (必需)
         "bmi": "22.5",                // 字符串或浮点数，体质指数。"0.00" 或 0 表示无数据
         "bmiStatus": "normal",        // 字符串: normal|overweight|underweight|obese
         "cardioLevel": "良好",        // 字符串，心肺功能等级描述。"未评估"表示无数据
         "cardioStatus": "good",       // 字符串: excellent|good|normal|poor

             "title": "增加有氧运动",         // 建议标题
             "description": "建议每周进行3-5次有氧运动...",  // 详细描述

================================================================================
                        三、数据计算逻辑（前端实现）
================================================================================

总体健康评分 = (运动评分 + 饮食评分 + 睡眠评分) / 3

范围: 0-100
显示: 圆形进度条（SVG），圆周率为 54，线宽为 8

================================================================================
                        四、指标说明
================================================================================

【运动评分】 (exerciseScore)
- 来源: 用户运动签到数据
- 范围: 0-100
- 影响因素: 每周运动频率、运动时长、运动强度
- 标准: 
  * 80-100: 每周运动5次以上，每次30分钟以上
  * 60-79: 每周运动3-4次
  * 40-59: 每周运动1-2次
  * 0-39: 基本无运动或运动不规律

【饮食评分】 (mealScore)
- 来源: 用户饮食签到数据
- 范围: 0-100
- 影响因素: 饮食规律、营养均衡、卡路里控制
- 标准:
  * 80-100: 三餐规律，营养均衡，无高油高盐高糖食物
  * 60-79: 基本三餐规律，偶有不均衡
  * 40-59: 不规律进食，营养结构差
  * 0-39: 饮食习惯极差

【睡眠评分】 (sleepScore)
- 来源: 用户睡眠签到数据
- 范围: 0-100
- 影响因素: 睡眠时长、睡眠规律性、入睡时间
- 标准:
  * 80-100: 7-8小时睡眠，23:00-07:00规律睡眠
  * 60-79: 6-7小时睡眠，基本规律
  * 40-59: 5-6小时或不规律
  * 0-39: 严重睡眠不足或极不规律

【BMI 指数】 (bmi)
- 计算: weight(kg) / height(m)²
- 范围: 通常 15-40
- 状态映射:
  * BMI < 18.5: underweight (消瘦)
  * 18.5 ≤ BMI < 24: normal (正常)
  * 24 ≤ BMI < 28: overweight (超重)
  * BMI ≥ 28: obese (肥胖)

【心肺功能】 (cardioLevel, cardioStatus)
- 来源: 根据运动数据评估
- 等级: excellent|good|normal|poor
- 评估标准: 
  * excellent: 有氧运动能力强，运动评分 ≥ 85
  * good: 有氧运动能力较好，运动评分 70-84
  * normal: 有氧运动能力一般，运动评分 50-69
  * poor: 有氧运动能力弱，运动评分 < 50

【代谢指数】 (metabolism)
- 范围: 0-100
- 计算: 基于 BMI、年龄、性别、运动量
- 公式建议: (当前BMI适度值 * 100) / 参考BMI值
- 标准:
  * 85-100: 代谢高效
  * 70-84: 代谢正常
  * 50-69: 代谢偏低
  * 0-49: 代谢低下

【睡眠质量】 (sleepQuality, sleepQualityStatus)
- 来源: 根据睡眠签到数据
- 等级: excellent|good|normal|poor
- 映射关系:
  * excellent: 睡眠评分 ≥ 90
  * good: 睡眠评分 75-89
  * normal: 睡眠评分 50-74
  * poor: 睡眠评分 < 50

【压力管理指数】 (用于雷达图)
- 范围: 0-100
- 计算建议: 
  * 如有压力测试数据，直接使用
  * 如无数据，根据运动频率和睡眠质量推导:
    压力指数 = (运动评分 * 0.4 + 睡眠评分 * 0.6)
- 标准:
  * 70-100: 压力管理良好，心态平和
  * 50-69: 压力适中，需要调整
  * 0-49: 压力大，需立即改善

================================================================================
                    五、建议生成逻辑（后端实现参考）
================================================================================

根据用户当前的各项指标生成个性化建议，至少需要 4 条不同的建议：

建议优先级判断:
- 【高】: 该指标评分 < 60，或与理想值相差 > 20
- 【中】: 该指标评分 60-75，或与理想值相差 10-20
- 【低】: 该指标评分 > 75，或与理想值相差 < 10

建议示例:

运动建议（当运动评分 < 70时）:
- 高: "增加有氧运动"
- 中: "加强运动计划执行力"
- 低: "保持现有运动习惯"

饮食建议（当饮食评分 < 70时）:
- 高: "均衡膳食结构，增加蔬菜水果"
- 中: "改善饮食习惯，减少高热量食物"
- 低: "维持现有饮食水平"

睡眠建议（当睡眠评分 < 80时）:
- 高: "规律作息，22:30前入睡"
- 中: "逐步调整睡眠时间"
- 低: "保持良好睡眠习惯"

其他建议:
- 心肺功能: "增加有氧运动强度"（当 cardioStatus 不是 excellent）
- 压力管理: "学习冥想和放松技巧"（当压力指数 < 60）
- 水分补充: "每天饮用8杯水"（固定建议）

================================================================================
                    六、时间轴数据生成逻辑（后端实现参考）
================================================================================

时间轴展示用户的健康进度里程碑，应根据实际数据生成，建议包括:

1. 档案初始化事件（用户首次完成档案）
   - date: 档案完成日期
   - title: "健康数据初始化"
   - description: "完成基础信息和体检数据录入，建立个人健康档案"
   - status: "completed"

2. 各维度达成事件（当某个评分首次达到特定阈值）
   - 运动评分达到 70+
   - 饮食评分达到 70+
   - 睡眠评分达到 80+

3. 综合成就事件
   - 总体评分达到 75+
   - 连续7天打卡
   - 各维度均衡达成

4. 当前进行中的事件
   - 某个维度的月度或季度目标
   - status: "in-progress"

示例:
{
  "date": "2024年3月",
  "title": "运动习惯养成",
  "description": "连续30天运动打卡，运动评分从60提升到75",
  "status": "completed"
}

4. 测试场景：

   - 数据加载中 → 显示加载动画

5. 缓存策略（可选）：
   - 建议在 30 分钟内缓存数据，减少 API 调用
   - 用户主动刷新或关闭后重新打开时，重新加载

================================================================================
                    八、前端无数据处理策略（自动启用）
================================================================================

当用户数据不完整或为 0 时，前端会自动使用以下策略确保良好的用户体验：

【评分显示】
- 如果评分为 0: 显示 "--"，表示 "待评分"
- 如果评分 > 0: 正常显示数字评分

【身体指标卡片】
- BMI 为 0: 显示 "待更新"，提示 "请完成基础信息设置"
- 心肺功能未评估: 显示 "坚持运动即可提升"
- 代谢指数为 0: 显示 "待评估"，提示 "记录运动数据后评估"
- 睡眠质量为 0: 显示 "待评估"，提示 "开始记录睡眠数据"

【建议列表】
如果后端未返回建议或返回空数组，使用以下默认建议：
- 🏃 增加有氧运动 (高)
- 🥗 均衡膳食结构 (高)
- 🌙 规律作息 (高)
- 💧 适当补水 (低)

【时间轴】
如果后端未返回时间轴或返回空数组，使用以下默认时间轴：
- 健康档案已建立 (当前日期, completed)
- 运动习惯养成 (即将开始, in-progress)
- 饮食质量改善 (即将开始, pending)
- 睡眠质量优化 (即将开始, pending)

【雷达图】
根据用户的基本评分动态生成：
- 运动: exerciseScore
- 饮食: mealScore
- 睡眠: sleepScore
- 心肺: 根据 cardioStatus 映射 (excellent:85, good:70, normal:50, poor:30)
- 代谢: metabolism
- 压力管理: (exerciseScore * 0.4 + sleepScore * 0.6)

================================================================================
                        八、错误处理
================================================================================

前端会处理以下错误场景：

1. 用户未登录
   - HTTP 401: 重定向到登录页

2. 用户未完成档案设置
   - API 返回 code: 1，msg: "用户未完成健康档案设置"
   - 显示强制设置弹窗

3. 网络错误或超时
   - 显示错误提示，允许用户重试

4. 服务器错误 (500)
   - 显示通用错误提示，建议用户稍后重试

================================================================================
                        十一、常见问题 FAQ
================================================================================

Q1: 如果用户数据不完整怎么办？
A: 使用合理的默认值或 0 值，前端会正常展示，但建议在返回消息中提示用户补充数据。

Q2: 为什么需要那么多维度？
A: 多维度展示能提供更全面的健康画像，帮助用户了解自己的健康状况。

Q3: 时间轴数据应该由前端还是后端生成？
A: 强烈建议后端生成，因为后端掌握完整的用户数据历史。

Q4: 建议需要实时生成还是可以预先生成？
A: 建议实时生成更好，这样用户看到的建议总是最新且个性化的。

Q5: API 返回速度有要求吗？
A: 建议在 500ms 内返回，以确保良好的用户体验。

Q6: 当用户没有任何数据时怎么显示？
A: 前端会自动使用友好的默认值和提示信息:
   - 评分为 0 时显示 "--"
   - BMI 为 0 时显示 "待更新" 和 "请完成基础信息设置"
   - 心肺功能为 "未评估" 时显示 "坚持运动即可提升"
   - 建议和时间轴如果为空，使用预设的默认值
   这样确保用户体验流畅，同时引导用户完成数据填充

Q7: 优先级是中文还是英文？
A: 后端应返回英文优先级 (high|medium|low)，前端会自动转换为中文显示 (高|中|低)
